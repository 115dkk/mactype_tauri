# Visual target specification

The target is an authored layout sketch derived from the supplied blueprint rather than an imitation of the unavailable Delphi frontend.

- 1180 by 760 desktop window with a 220 pixel left rail and flexible workspace.
- Product lockup, three primary destinations, and theme control in the rail.
- Overview uses one warning band, a bordered definition list, and a flat next-action row.
- Profile uses a narrow settings index, line-separated form rows, and a fixed inline preview below the form.
- Diagnostics uses a component definition list and dense three-column log rows.
- Mobile changes the rail into a horizontal toolbar, stacks setting controls, and preserves all functions.
- Type scale is 24, 16, 14, 13, and 12 pixels. Only the view name uses 24 pixels.
- Radius is 4 pixels throughout. There are no in-window shadows, gradients, decorative metrics, or floating cards.
- Light and dark themes retain the same hierarchy and do not invert individual sections.
